# Руководство по устранению проблем

Распространённые проблемы и решения для RPAForge.

## Проблемы установки

### Установка зависимостей Python завершается ошибкой

**Симптом**: `pip install` завершается ошибками зависимостей.

**Решение**:
```bash
# Сначала обновите pip
pip install --upgrade pip

# Установите со всеми зависимостями
pip install rpaforge-core rpaforge-libraries[all]

# Или используйте uv для более быстрой установки
uv pip install rpaforge-core rpaforge-libraries[all]
```

### Модуль cryptography не найден

**Симптом**: `ModuleNotFoundError: No module named 'cryptography'`

**Решение**:
```bash
pip install cryptography
```

### Ошибки импорта pywinauto в Windows

**Симптом**: Не удаётся инициализировать автоматизацию рабочего стола.

**Решение**:
```bash
# Установите Visual Studio Build Tools, если они отсутствуют
# Скачать можно здесь: https://visualstudio.microsoft.com/downloads/

# Затем установите pywinauto
pip install pywinauto
```

## Проблемы выполнения

### Studio не запускается

**Симптом**: Приложение Electron не удаётся запустить.

**Решения**:
1. Проверьте версию Node.js ( требуется 20+):
   ```bash
   node --version
   ```

2. Пересоберите нативные модули:
   ```bash
   cd packages/studio
   rm -rf node_modules package-lock.json
   pnpm install
   ```

3. Проверьте конфликты портов:
   ```bash
   lsof -i :9222  # Порт отладки по умолчанию
   ```

### Таймаут выполнения процесса

**Симптом**: Действия достигают таймаута, даже когда должны завершиться.

**Решения**:
1. Увеличьте таймаут в настройках действия
2. Проверьте доступность системных ресурсов
3. Проверьте размер пула субпроцессов:
   ```python
   from rpaforge.core.subprocess_executor import get_pool_stats
   print(get_pool_stats())
   ```

### Хранилище учётных данных заблокировано

**Симптом**: `ValueError: Vault is locked` при доступе к учётным данным.

**Решения**:
1. Используйте менеджер контекста для правильной очистки:
   ```python
   with Credentials() as cred:
       cred.store_credential("key", "user", "pass")
   ```

2. Разблокируйте вручную при необходимости:
   ```python
   cred = Credentials()
   # Хранилище автоматически разблокируется при первом доступе
   ```

## Проблемы автоматизации

### DesktopUI: Элемент не найден

**Симптом**: `ElementNotFoundError` при работе с UI-элементами.

**Решения**:
1. Используйте Инспектор селекторов для проверки селектора:
   - Нажмите `Ctrl+Shift+S` для открытия Инспектора селекторов
   - Наведите курсор на элемент для просмотра его свойств

2. Попробуйте разные стратегии поиска:
   - Используйте `title` вместо `class_name`
   - Добавьте параметр `timeout`

3. Увеличьте время ожидания:
   ```python
   ui.wait_for_element("selector", timeout="10s")
   ```

### Database: Не удаётся подключиться к базе данных

**Симптом**: Не удаётся подключиться к базе данных.

**Решения**:
1. Проверьте формат строки подключения:
   ```python
   # SQLite
   db.connect(":memory:")  # В памяти
   db.connect("path/to/db.sqlite")  # Файл
   
   # PostgreSQL
   db.connect("postgresql://user:pass@host:5432/dbname")
   ```

2. Проверьте, запущен ли сервер базы данных
3. Проверьте правила брандмауэра для удалённых подключений

### WebUI: Браузер не запускается

**Симптом**: Браузер Playwright не удаётся запустить.

**Решения**:
1. Установите бинарные файлы браузера:
   ```bash
   playwright install chromium
   ```

2. Проверьте наличие системных зависимостей (Linux):
   ```bash
   sudo apt-get install libnss3 libnspr4 libatk-bridge2.0-0 libdrm2 libxkbcommon0 libgbm1
   ```

## Проблемы производительности

### Медленное выполнение процесса

**Симптом**: Автоматизация работает медленнее ожидаемого.

**Решения**:
1. Уменьшите накладные расходы пула субпроцессов:
   ```python
   executor = SubprocessExecutor(max_workers=2)
   ```

2. Используйте параллельное выполнение для независимых задач:
   ```python
   # Используйте блок Parallel для параллельных действий
   ```

3. Отключите отладку для продакшена:
   - Отключите проверки точек останова
   - Уменьшите подробность логов

### Высокое использование памяти

**Симптом**: RPAForge потребляет слишком много RAM.

**Решения**:
1. Ограничьте пул субпроцессов:
   ```python
   env["RPAFORGE_MAX_WORKERS_LIMIT"] = "4"
   ```

2.正确но закрывайте ресурсы:
   ```python
   with Credentials() as cred:
       # Используйте учётные данные
   # Автоматическая очистка при выходе
   ```

## Конфигурация

### Переменные среды

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `RPAFORGE_MAX_WORKERS_LIMIT` | `cpu_count * 4` | Максимальный размер пула субпроцессов |
| `RPAFORGE_LOG_LEVEL` | `INFO` | Уровень логирования |
| `RPAFORGE_LOG_DIR` | `~/.rpaforge/logs` | Директория файлов логов |

### Пользовательский путь к библиотекам

**Симптом**: Пользовательские RPA-библиотеки не найдены.

**Решение**:
```bash
export PYTHONPATH="/path/to/custom/libraries:$PYTHONPATH"
```

## Получение помощи

Если проблемы сохраняются:

1. Проверьте [GitHub Issue](https://github.com/chelslava/rpaforge/issues)
2. Запустите с логированием отладки:
   ```bash
   export RPAFORGE_LOG_LEVEL=DEBUG
   rpaforge ...
   ```
3. Соберите логи из `~/.rpaforge/logs/`
